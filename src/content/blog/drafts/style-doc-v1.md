## Typography

### Font Stack

```css
font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', 'Roboto', 
             'Helvetica Neue', Arial, sans-serif;
```

### Scale & Hierarchy

<div style="font-size: 2.25rem; font-weight: 700; line-height: 1.2; margin: 1rem 0;">H1 - Main Headings</div>

- **Size**: 36px / 2.25rem
- **Weight**: 700 (Bold)
- **Line Height**: 1.2
- **Usage**: Page titles, primary headings

<div style="font-size: 1.875rem; font-weight: 600; line-height: 1.3; margin: 1rem 0;">H2 - Section Headings</div>

- **Size**: 30px / 1.875rem
- **Weight**: 600 (Semi-bold)
- **Line Height**: 1.3
- **Usage**: Major section dividers

<div style="font-size: 1.5rem; font-weight: 600; line-height: 1.4; margin: 1rem 0;">H3 - Subsection Headings</div>

- **Size**: 24px / 1.5rem
- **Weight**: 600 (Semi-bold)
- **Line Height**: 1.4
- **Usage**: Subsections, article headings

<div style="font-size: 1rem; font-weight: 400; line-height: 1.6; margin: 1rem 0;">Body Text - Regular paragraphs and content</div>

- **Size**: 16px / 1rem
- **Weight**: 400 (Regular)
- **Line Height**: 1.6
- **Usage**: All body content, default text

<div style="font-size: 0.875rem; font-weight: 400; line-height: 1.5; margin: 1rem 0; color: #6b7280;">Small Text - Captions, metadata, fine print</div>

- **Size**: 14px / 0.875rem
- **Weight**: 400 (Regular)
- **Line Height**: 1.5
- **Color**: Text Light (`#6b7280`)
- **Usage**: Dates, captions, secondary info

## Spacing

### Scale

Based on 8px base unit for consistent rhythm:

- **xs**: 4px (0.25rem)
- **sm**: 8px (0.5rem)
- **md**: 16px (1rem)
- **lg**: 24px (1.5rem)
- **xl**: 32px (2rem)
- **2xl**: 48px (3rem)
- **3xl**: 64px (4rem)

### Usage

- **Component padding**: sm to md
- **Section spacing**: xl to 2xl
- **Page margins**: lg to xl
- **Element gaps**: xs to sm

## Layout

### Container Widths

- **Mobile**: Full width with 16px side padding
- **Tablet**: 768px max width
- **Desktop**: 1024px max width
- **Wide**: 1200px max width (for special layouts)

### Grid

- **Columns**: Flexible, typically 12-column system
- **Gaps**: 24px (lg) between columns
- **Breakpoints**: 640px (mobile), 768px (tablet), 1024px (desktop)

## Components

### Links

- **Default**: Primary color (`#2563eb`), no underline
- **Hover**: Slightly darker, subtle underline appears
- **Visited**: Slightly muted primary color
- **Focus**: Outline for keyboard navigation

<p><a href="#" style="color: #2563eb; text-decoration: none;">Example link styling</a></p>

### Buttons

<button style="background-color: #2563eb; color: white; padding: 8px 16px; border: none; border-radius: 6px; font-weight: 500; cursor: pointer; margin: 4px;">Primary Button</button>

<button style="background-color: transparent; color: #2563eb; padding: 8px 16px; border: 2px solid #2563eb; border-radius: 6px; font-weight: 500; cursor: pointer; margin: 4px;">Secondary Button</button>

### Code Blocks

```css
/* Inline code styling */
code {
  background-color: #f1f5f9;
  padding: 2px 6px;
  border-radius: 4px;
  font-family: 'SF Mono', Monaco, 'Cascadia Code', monospace;
  font-size: 0.875em;
}
```

Block code uses syntax highlighting with a subtle background and consistent padding.

### Blockquotes

> This is how quotes appear - with a subtle left border, italic text, and proper spacing to distinguish them from regular content.

## Content Guidelines

### Voice & Tone

- **Conversational but professional** - like talking to a smart colleague
- **Direct and clear** - avoid unnecessary jargon or fluff
- **Helpful and genuine** - prioritize usefulness over cleverness
- **Confident but not arrogant** - share knowledge without talking down

### Writing Style

- **Sentence length**: Mix short and medium sentences
- **Paragraphs**: 2-4 sentences typically, break up long blocks
- **Headers**: Use sentence case, not title case
- **Lists**: Introduce with a colon, parallel structure

### Images

- **Format**: WebP preferred, JPG fallback
- **Sizing**: Responsive, maximum 1200px width
- **Alt text**: Descriptive, not just decorative
- **Captions**: When needed, use small text styling

## Accessibility

### Standards

- **WCAG 2.1 AA compliance** minimum target
- **Color contrast**: 4.5:1 for normal text, 3:1 for large text
- **Focus indicators**: Visible for keyboard navigation
- **Alt text**: Meaningful descriptions for all images
- **Semantic HTML**: Proper heading hierarchy, landmarks

### Testing

- **Screen readers**: Test with VoiceOver/NVDA periodically
- **Keyboard navigation**: All interactive elements accessible
- **Color blindness**: Ensure information isn't color-dependent

## Technical Notes

### CSS Architecture

- **Methodology**: Utility-first with component classes
- **Naming**: BEM for components, descriptive utilities
- **Organization**: Base → Components → Utilities → Pages

### Performance

- **Critical CSS**: Inline above-the-fold styles
- **Font loading**: `font-display: swap` for web fonts
- **Images**: Lazy loading, appropriate sizing
- **Minification**: CSS and JS compressed in production
